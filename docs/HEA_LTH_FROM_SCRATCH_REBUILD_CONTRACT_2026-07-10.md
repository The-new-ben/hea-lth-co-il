# Hea-lth from-scratch rebuild contract

Date: 2026-07-10

This is the operating contract for rebuilding `hea-lth.co.il`.

## Non-negotiable direction

- Do not preserve the current website design.
- Do not patch the current homepage.
- Do not stack cosmetic layers on top of the old WordPress presentation.
- Build a new WordPress theme system from source control.
- Keep only strategic assets: domain history, URL equity, GSC evidence, existing slugs that have search value, and any audited content/data worth migrating.
- Every important page must belong to a governed hierarchy: pillar, cluster, spoke, glossary/entity, provider profile, price guide, or conversion route.
- Every article/page must have one primary query intent and one canonical URL. No uncontrolled cannibalization.
- Full long-form medical articles are not generated directly inside Codex. Codex creates the research protocol, SERP evidence, keyword map, article brief, prompt, review gates, metadata, schema plan, and publishing system.

## Product standard

Hea-lth is not a small brochure site. It is being rebuilt as a premium Hebrew health platform:

1. Consumer research and comparison experience.
2. Verified provider and clinic discovery.
3. Premium private-medicine, aesthetics, diagnostics, wellness, hair, plastic surgery, and medical-device commercial routes.
4. Professional acquisition and provider onboarding.
5. Governed lead routing with consent, privacy boundaries, and no unsafe diagnosis collection.
6. Future-grade capabilities: interactive maps, AI navigation, 3D anatomy/treatment education, provider dashboards, comparison tools, CRM integrations, and data products.

## SEO preservation rule

The old website can be replaced. The old URL value cannot be casually destroyed.

Before changing, deleting, redirecting, or canonicalizing an existing URL:

1. Check GSC evidence.
2. Check crawl/index state.
3. Check backlinks if available.
4. Check SERP intent.
5. Check keyword overlap with candidate target pages.
6. Decide: keep-improve, merge, redirect, noindex, or rebuild under same slug.

## Design/build rule

The new build must be theme-first:

- New theme files in Git.
- New homepage template.
- New header, footer, navigation, template hierarchy, design tokens, and responsive system.
- New conversion forms and routing logic.
- New templates for hubs, money pages, article pages, provider profiles, and professional/account areas.
- Existing live WordPress is only the host and SEO asset holder until the new theme is ready to activate.

## Proof standard

Every material step must be backed by one or more of:

- live browser screenshot,
- Git diff,
- Google Search Console / keyword data,
- SERP capture,
- competitor benchmark,
- official documentation source,
- automated test or deploy run,
- production endpoint verification.
